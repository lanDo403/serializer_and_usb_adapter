# Система высокоскоростного обмена данными на базе ПЛИС с использованием интерфейса USB 3.0

## Обзор

Проект реализует тракт обмена данными на базе `Xilinx Spartan-6` и `FTDI FT601` в режиме `245 synchronous FIFO`.

Основной сценарий работы:

- принять 8-битный поток данных по GPIO;
- упаковать байты в 32-битные слова;
- буферизовать поток в FIFO;
- передать данные на ПК через `USB 3.0`.

Дополнительно предусмотрен тестовый режим loopback для проверки интерфейса FT601 без внешнего источника на GPIO.

Этот файл является одновременно описанием проекта и рабочей спецификацией RTL. При верификации следует опираться на него.

## Потоки данных

Основной режим:

`GPIO -> get_gpio -> packer8to32 -> fifo_tx -> fifo_fsm -> FT601 -> ПК`

Тестовый loopback-режим:

`ПК -> FT601 -> fifo_rx -> fifo_fsm -> FT601 -> ПК`

Loopback нужен для отладки FT601 и не является основным пользовательским режимом.

## Верхний модуль

Текущий верхний модуль:

- `fpga/top.v`

Внешние интерфейсы:

- GPIO:
  - `GPIO_CLK`
  - `GPIO_DATA[7:0]`
  - `GPIO_STROB`
  - `FPGA_RESET`
- FT601:
  - `CLK`
  - `RESET_N`
  - `TXE_N`
  - `RXF_N`
  - `OE_N`
  - `WR_N`
  - `RD_N`
  - `BE[3:0]`
  - `DATA[31:0]`

Верхний модуль содержит только внешние интерфейсы и соединение внутренних блоков.

## Тактовые домены

В проекте используются два домена.

### GPIO-домен

- источник такта: `GPIO_CLK`
- расчетная частота: до `50 МГц`
- основные блоки:
  - `get_gpio.v`
  - `packer8to32.v`
  - write-сторона `fifo_dualport.v`
  - write-сторона `sram_dualport.v`
  - `fifo_tx_ctrl.v`

### FT601-домен

- источник такта: `CLK` от FT601
- текущая целевая частота: `66.67 МГц`
- основные блоки:
  - read-сторона `fifo_dualport.v`
  - read-сторона `sram_dualport.v`
  - RX FIFO
  - `fifo_fsm.v`
  - `fifo_rx_ctrl.v`
  - `ft601_io.v`

## Сброс

Внутренние сигналы сброса синхронизированы по доменам и активны низким уровнем.

Внешние источники сброса:

- `FPGA_RESET`
- `RESET_N`

Схема работы:

- `FPGA_RESET` буферизуется на верхнем уровне;
- `RESET_N` буферизуется в `ft601_io.v`;
- синхронизация сбросов по доменам выполняется в `rst_sync.v`;
- внутренняя логика использует:
  - `gpio_rst_n_i`
  - `ft_rst_n_i`

## Основные модули

### `fpga/get_gpio.v`

Захват входного GPIO-потока.

Функции:

- буферизация `GPIO_CLK`, `GPIO_DATA`, `GPIO_STROB`;
- регистрация входных байтов и строба;
- формирование локального интерфейса для упаковщика.

### `fpga/packer8to32.v`

Упаковка четырех валидных байтов в одно 32-битное слово.

Правила:

- байт принимается только при `valid_i = 1`;
- порядок байтов в слове:
  - первый -> `[7:0]`
  - второй -> `[15:8]`
  - третий -> `[23:16]`
  - четвертый -> `[31:24]`
- `valid_o` активен один такт при готовности полного слова.

### `fpga/fifo_dualport.v`

Асинхронный FIFO с раздельными тактами записи и чтения.

Функции:

- передача данных между GPIO и FT601 доменами;
- формирование `full` и `empty`;
- фиксация sticky `overflow` и опционального `underflow`.

Примечания:

- используются бинарные и Gray-указатели;
- синхронизация указателей выполняется двухтактными синхронизаторами;
- данные хранятся во внешнем `sram_dualport.v`.

### `fpga/sram_dualport.v`

Двухпортовая синхронная память для FIFO.

Чтение синхронное, поэтому при проверке read-path нужно учитывать задержку памяти.

### `fpga/fifo_tx_ctrl.v`

Логика управления записью в TX FIFO.

Функции:

- разрешение записи из `packer8to32`;
- блокировка новых слов после sticky TX или RX ошибок;
- прием сигнала очистки TX ошибки через синхронизированный toggle.

### `fpga/fifo_rx_ctrl.v`

Логика обработки слов, полученных от FT601.

Функции:

- дренирование RX FIFO в normal mode;
- фиксация sticky RX и TX ошибок;
- декодирование служебных команд от хоста.

Правило режимов:

- при `FT_LOOPBACK_TEST = 1'b0` входящие RX слова трактуются как команды;
- при `FT_LOOPBACK_TEST = 1'b1` `fifo_rx_ctrl` не владеет RX FIFO, а слова трактуются как payload для loopback.

Поддерживаемые команды при `BE = 4'hF`:

- `32'h00000001` - очистить TX error
- `32'h00000002` - очистить RX error
- `32'h00000003` - очистить обе ошибки

### `fpga/fifo_fsm.v`

Конечный автомат интерфейса FT601.

Функции:

- арбитраж между TX и RX;
- генерация `WR_N`, `RD_N`, `OE_N`;
- управление `fifo_pop` и `fifo_append`;
- передача `DATA` и `BE` через `ft601_io.v`.

Текущий набор состояний:

- `ARB`
- `TX_PREFETCH`
- `TX_BURST`
- `RX_START`
- `RX_BURST`

### `fpga/ft601_io.v`

Физическая обвязка FT601.

Функции:

- `IBUFG` для `CLK`;
- `IBUF` для `RESET_N`, `TXE_N`, `RXF_N`;
- `OBUF` для `OE_N`, `WR_N`, `RD_N`;
- `IOBUF` для `DATA[31:0]` и `BE[3:0]`.

Этот модуль отделяет pin-level обвязку от логики протокола.

### `fpga/rst_sync.v`

Синхронизатор сброса для домена.

Функции:

- принимает асинхронный запрос на сброс;
- снимает сброс синхронно с локальным тактом;
- выдает active-low reset для внутренней логики.

## Режимы работы

### Normal mode

Управляется параметром:

- `FT_LOOPBACK_TEST = 1'b0`

Поведение:

- данные приходят с GPIO;
- слова упаковываются и записываются в TX FIFO;
- FT601 читает TX FIFO и отправляет поток на ПК;
- RX канал FT601 используется только для служебных команд.

Примечания:

- входящие слова FT601 в этом режиме не считаются произвольным payload;
- значения `1`, `2`, `3` при `BE = 4'hF` являются командами, а не пользовательскими данными;
- передача случайного payload с ПК в этом режиме не является штатным сценарием.

### FT loopback mode

Управляется параметром:

- `FT_LOOPBACK_TEST = 1'b1`

Поведение:

- хост пишет данные в FT601;
- FPGA принимает слова в RX FIFO;
- этот же RX FIFO становится источником TX для передачи обратно хосту;
- RX данные трактуются как payload, а не как команды.

Примечания:

- входные данные loopback могут быть произвольными;
- `fifo_rx_ctrl` не читает RX FIFO в этом режиме;
- режим используется для проверки FT601 RX/TX handshake и общей связности интерфейса.

## Обработка ошибок

В проекте используются sticky-ошибки.

### TX error

Фиксируется, если:

- выполнена попытка записи в полный TX FIFO;
- обнаружен некорректный TX-side доступ в FT-домене.

После фиксации:

- новые записи из `packer8to32` в TX FIFO блокируются;
- очистка выполняется только командой от хоста.

### RX error

Фиксируется, если:

- произошел overflow RX FIFO;
- произошел underflow RX FIFO, если он включен для данного экземпляра FIFO.

Очистка также выполняется только командой от хоста.

## Верификация

Основной testbench:

- `fpga/testbench.v`

Что проверяется:

- сброс в обоих доменах;
- согласованность параметра `FT_LOOPBACK_TEST`;
- normal mode `GPIO -> FT601`;
- loopback mode `FT601 -> FPGA -> FT601`;
- безопасное состояние FT интерфейса во время reset;
- совпадение переданных слов с ожидаемыми, построенными из `fpga/data_p`.

Testbench должен запускаться в двух вариантах:

1. `FT_LOOPBACK_TEST = 0`
2. `FT_LOOPBACK_TEST = 1`

## Текущие ограничения

- основной рабочий тракт односторонний: `GPIO -> FT601 -> ПК`
- loopback является тестовым режимом, а не пользовательской функцией
- старт TX пока медленнее, чем в reference design FTDI
- в reference FTDI TX prefetch реализован чище, чем в текущем локальном варианте
- для GPIO входа задано ограничение по периоду такта, но нет формализованного внешнего timing contract для `GPIO_DATA` и `GPIO_STROB`
